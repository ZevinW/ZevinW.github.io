中间件关键字

是一种装配到应用管道以处理请求和响应的软件。 每个组件：

  • 选择是否将请求传递到管道中的下一个组件。
  • 可在管道中的下一个组件前后执行工作。

请求委托用于生成请求管道。请求委托处理每个HTTP请求。

使用RunMap 和 Use 扩展方法来配置请求委托。

可将一个单独的请求委托并行指定为匿名方法（称为并行中间件），或在可重用的类

中对其进行定义。这些可重用的类和并行匿名方法即为中间件，也叫中间件组件。

请求管道中的每个中间件组件负责调用管道中的下一个组件，或使管道短路。当中间件短路时，它被称为“终端中间件”，因为它阻止中间件进一步处理请求。

Run 的使用 （终结点）

Run:　使管道短路，Run后的代码都不会被执行，所以也叫终节点中间件，这就是为什么 app.Run() ;

要放到最后面的原因。

注： 即便在框架内置中间件前 添加了 app.Run() 的代码。调试时也是先执行内置的中间件

原因：.NET 会在 CRL 这里对代码进行优化。即便把 代码也在了上面，但是CRL 会对你这个代码进行优化。

Use 的 使用 （短路）

可以使管道短路

通过不调用 next.Invoke 实现短路

注：
通过 next.Invoke来请求下一个管道，通过注释 next.Invoke来让管道短路

Map 的 使用 （定制路由）

UseWhen 的 使用（当符合批定的条件时执行，他不会替代主管道）

注：通过 useWhen 可以结合 Use 或是 Run 执行更复杂的逻辑，可以更多的条件判断与处理

它不会替代主管道，所以不会影响后面管道的运行

MapWhen 的 使用 （当符合批定的条件时执行，他会替代主管道）

注： 通过 MapWhen 可以结合 use 或是 Run 执行更复杂的逻辑

可以更多的条件判断与处理

它会替代主管道，会影响后续的程序执行。

系统默认中间件的顺序

UseCors , UseAuthentication 和 UseAuthorization 必须按显示的顺序出现。

UseCors 当前必须在 UseReponseCaching之前出现。

UseRequestLocalization必须在可能检查请求区域性的任务中间件 （例如 app.UseStaticFiles()）之前出现。

在使用 速率限制终结点特定的 API 时，必须在 UseRouting() 之后调用 UseRateLimiter.

中间件和过滤器有什么不同

过滤器和中间件都能实现记录异常，记录用户请求和响应参数等功能。

但是过滤器主要实现的是功能，它是系统提供的特定的处理一些功能的。

但是中间件不一样，它是一个业务软件，比如跨域，静态文件中间件等，它强调一个类似

业务软件的，它能应用于不同的系统。
